Learning Experience

This project was created during the Harvard (CS50) computer science certificate. I was pretty new to coding at this point and looking back I probably took on a project that was a little too ambitious.

This was my first time working with a full development stack. The CS50 course only covered the absolute basics of the flask microframework and did not cover the SQLAlchemy at all except for an short guide included in the 'how to deploy your project to Heroku' section.

Javascript

I first wrote the code for this website in vanilla JavaScript and I have to say DOM manipulation was a long and laborious process. I also initially created functions in the global scope polluting the window object. I refactored my code after following the Pluralsight JavaScript Practical Design Patterns course to improve the code quality.

At the same time as refactoring I decided to implement JQuery to clean up my code even further as this made DOM manipulation much less verbose.

Python

General Python

Whilst my Python code for this project does function as required I do feel that now I am more experienced I would write it differently were I to create a similar project. The routes are too long and should be calling functions to carry out tasks. That said to achieve the functionality that I have with this app I am still proud of what it can do.

Flask

This was taught on the CS50 course and as such I found development using Flask quite simple compared to some other aspects of this project. Writing logic for the login, register and test pathways created the primary difficulties due to working with database queries.

To critisise my app in some capacity I would say that the templating could be improved by better nesting of HTML elements especially areas where there are large areas of overlap in layout. Although with modern frontend frameworks such as Angular and React taking the need for templating away from more modern applications this would be handled outside of the Flask system.

SQLAlchemy

Let me start with this, trying to teach yourself an ORM from just the documentation is not for the faint hearted. I had many failures on my way to success with SQLAlchemy much of which was the result of my failing to correctly interpret the documentation the first time I read it requiring me to go back and reread it again.

Needless to say I persisted and am now a lot more comfortable building and using SQLAlchemy. I have since used it twice more as I am following the Harvard web development programme

CSS

Flexbox

The original implementation of this app was via the flexbox layout module. The shorter projects that I had created up until this point had just used CSS margin/padding to position elements on the screen and so I wanted to try something new and Flexbox I felt was my next step. Although Flexbox does offer good functionality having learnt about CSS Grid I felt that the grid system is vastely easier to implement and therefore I elected to change the layout to CSS Grid.
